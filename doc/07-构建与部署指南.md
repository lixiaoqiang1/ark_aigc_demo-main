## 1. 部署前端

### 1.1 本地构建

在项目根目录：

```powershell
npm run build
```

产物在本地 `build/`（`index.html`、`static/` 等）。  
生产环境 API 走同域（`AIGC_PROXY_HOST = ''`），由 Nginx 反代到后端 `3001`。

### 1.2 上传到服务器

**方式 A：一键脚本（推荐）**

```powershell
# 需先设置上面的 DEPLOY_* 环境变量
python publish.py --frontend
```

**方式 B：手动 zip + scp**

```powershell
Compress-Archive -Path "build\*" -DestinationPath "build.zip" -Force
scp -o StrictHostKeyChecking=no build.zip root@136.175.82.211:/tmp/build.zip

ssh root@136.175.82.211 "cd /var/www/ark_aigc_demo/build && rm -rf static asset-manifest.json favicon.png robots.txt index.html && unzip -o /tmp/build.zip -d /var/www/ark_aigc_demo/build/ && rm /tmp/build.zip && nginx -t && systemctl reload nginx && echo Frontend_OK"

Remove-Item build.zip -Force
```

### 1.3 前端部署后检查

- 浏览器打开：`https://136.175.82.211:3000/`（建议 Ctrl+Shift+R 强刷）
- 页面能打开、静态资源无 404 即表示前端已更新

## 2. 部署服务端（后端）

后端源码在本地 `server_python/`，线上目录为 `/var/www/ark_aigc_demo/server/`。

### 2.1 部署内容包含什么

| 内容 | 说明 |
|------|------|
| Python 代码 | `main.py`、`routers/`、`database.py` 等 |
| 依赖 | `requirements.txt` → 线上 `venv` 中 `pip install` |
| 场景配置 | `scenes/*.json`（线上 `Custom.json` 建议保留，勿被空模板覆盖） |
| 环境变量 | 线上 `server/.env`（如 `ARK_API_KEY`），部署脚本会尽量保留 |
| **不覆盖** | `server/data/`（数据库）、`.env`、已有 `scenes/Custom.json` |

### 2.2 一键部署后端

```powershell
$env:DEPLOY_HOST = "136.175.82.211"
$env:DEPLOY_USER = "root"
$env:DEPLOY_PASSWORD = "你的SSH密码"

python publish.py --backend
```

脚本会：打包上传 → 保留 `data/.env/Custom.json` → `pip install` → `systemctl restart ark-aigc-demo`。

### 2.3 手动部署后端（可选）

```powershell
# 本地打包（排除缓存与本地库）
tar -czf backend.tar.gz --exclude=__pycache__ --exclude=venv --exclude=.venv --exclude=data -C server_python .

scp -o StrictHostKeyChecking=no backend.tar.gz root@136.175.82.211:/tmp/backend.tar.gz
```

登录服务器执行：

```bash
BASE=/var/www/ark_aigc_demo

# 1) 备份数据库与密钥
mkdir -p /tmp/ark_preserve
cp -a $BASE/server/data /tmp/ark_preserve/ 2>/dev/null || true
cp -a $BASE/server/.env /tmp/ark_preserve/ 2>/dev/null || true
cp -a $BASE/server/scenes/Custom.json /tmp/ark_preserve/ 2>/dev/null || true

# 2) 解压代码到 server/（不要删掉整个 server，以免弄丢 venv）
tar -xzf /tmp/backend.tar.gz -C $BASE/server
# 若本地打包的是「server 目录本身」而非其内容，按实际结构调整路径

# 3) 还原保留项
mkdir -p $BASE/server/data $BASE/server/scenes
cp -a /tmp/ark_preserve/data/. $BASE/server/data/ 2>/dev/null || true
cp -a /tmp/ark_preserve/.env $BASE/server/.env 2>/dev/null || true
cp -a /tmp/ark_preserve/Custom.json $BASE/server/scenes/Custom.json 2>/dev/null || true

# 4) 安装依赖并重启
$BASE/server/venv/bin/pip install -r $BASE/server/requirements.txt
systemctl restart ark-aigc-demo
systemctl is-active ark-aigc-demo
```

> 更省事请直接用 `python publish.py --backend`（打包结构已对齐线上 `server/`）。

### 2.4 后端必备配置

文本对话接口 `/chat` 需要方舟密钥，任选其一：

1. 文件：`/var/www/ark_aigc_demo/server/.env`

2. 或写入场景：`server/scenes/Custom.json` → `VoiceChat.Config.LLMConfig.ApiKey`  
3. 并确认同文件中有 `LLMConfig.EndPointId`

改完后：

```powershell

```

### 2.5 后端常用命令

```powershell

```

## 3. 数据库（SQLite）

### 3.1 是什么、在哪里

| 项 | 说明 |
|----|------|
| 类型 | SQLite 单文件，**不是**远程可直连的 MySQL |
| 线上路径 | `/var/www/ark_aigc_demo/server/data/app.db` |
| 本地路径 | `server_python/data/app.db`（开发时自动生成） |
| 表 | `users`、`conversations`、`messages` 等（启动时 `init_db()` 自动建表） |

首次启动后端时，若不存在 `data/app.db`，服务会自动创建目录并建表，**一般无需单独「安装数据库」**。

### 3.2 部署时如何对待数据库

- **默认保留线上库**：`publish.py` / 手动步骤都会备份并还原 `server/data/`，避免用户与会话被覆盖。
- **不要**把本地开发用的 `app.db` 随便上传覆盖生产库。
- **不要**把 `app.db` 提交到 Git（已在 `.gitignore`）。

### 3.3 备份与恢复

**备份（下载到本机）：**

```powershell
scp -o StrictHostKeyChecking=no root@136.175.82.211:/var/www/ark_aigc_demo/server/data/app.db .\app.db.backup
```

**恢复（上传覆盖线上，慎用）：**

```powershell
# 建议先停服务，避免写入冲突
ssh root@136.175.82.211 "systemctl stop ark-aigc-demo"
scp -o StrictHostKeyChecking=no .\app.db.backup root@136.175.82.211:/var/www/ark_aigc_demo/server/data/app.db
ssh root@136.175.82.211 "systemctl start ark-aigc-demo"
```

**仅在服务器上再拷一份：**

```powershell
ssh root@136.175.82.211 "cp -a /var/www/ark_aigc_demo/server/data/app.db /var/www/ark_aigc_demo/server/data/app.db.bak.$(date +%Y%m%d)"
```

## 4. 在自己电脑上连接 / 查看远程数据库

因为是 SQLite 文件，**不能**像 MySQL 那样填「主机 + 端口 + 账号」直接连。推荐下面三种方式。

### 方式一：下载到本地再用图形工具打开（最简单）

1. 从服务器拉下库文件：

```powershell
# 建议先停一下写入，或接受「热拷」可能偶发不一致
scp -o StrictHostKeyChecking=no root@136.175.82.211:/var/www/ark_aigc_demo/server/data/app.db .\remote-app.db
```

2. 用任意 SQLite 客户端打开 `remote-app.db`，例如：
   - [DB Browser for SQLite](https://sqlitebrowser.org/)
   - VS Code 插件：SQLite / SQLite Viewer
   - DBeaver：新建连接 → SQLite → 选择该文件

3. 查表示例：

```sql
SELECT * FROM users LIMIT 20;
SELECT * FROM conversations ORDER BY updated_at DESC LIMIT 20;
SELECT * FROM messages ORDER BY created_at DESC LIMIT 50;
```

> 这是**副本**。你在本地改库**不会**自动同步回服务器；要回写需再 `scp` 上传（见 3.3，生产慎用）。

### 方式二：SSH 进服务器，用命令行查（不下载）

```powershell
ssh root@136.175.82.211
```

```bash
# 若未安装 sqlite3：apt-get update && apt-get install -y sqlite3
cd /var/www/ark_aigc_demo/server/data
sqlite3 app.db

.tables
.schema users
SELECT id, username, created_at FROM users;
.quit
```

一行命令示例：

```powershell
ssh root@136.175.82.211 "sqlite3 /var/www/ark_aigc_demo/server/data/app.db 'SELECT COUNT(*) FROM users;'"
```

### 方式三：SFTP / 资源管理器挂载后打开

用 WinSCP、FileZilla、Cursor/VS Code Remote-SSH 连上服务器，浏览到：

`/var/www/ark_aigc_demo/server/data/app.db`

下载后用本地工具打开（同方式一）。  
若用 Remote-SSH 直接在服务器上装扩展打开该文件，也相当于在远程读库。

### 不推荐 / 不适用

| 做法 | 原因 |
|------|------|
| 把 `3001` 端口暴露公网直连数据库 | 后端是 HTTP API，**不是**数据库端口；且库是文件 |
| Navicat/DBeaver 填 IP:3306 连 MySQL | 本项目没有 MySQL |
| 长时间打开线上 `app.db` 做写入调试 | 易与正在运行的 uvicorn 抢锁，可能损坏数据 |

若以后要改成「本机 GUI 实时连远程库」，需要迁移到 PostgreSQL/MySQL，再通过 SSH 隧道连接；**当前架构不支持**。

## 7. 注意事项

1. 只改前端：构建 + 上传 `build/` + `nginx reload` 即可。  
2. 只改后端：同步代码 + `pip install` + `systemctl restart ark-aigc-demo`。  
3. 部署后端时**务必保留**线上 `data/app.db`，否则用户数据会丢。  
4. 密钥（SSH 密码、`ARK_API_KEY`、场景里的 AK/SK）不要提交仓库。  
5. 不要占用/覆盖同机其它项目目录与端口；本项目对外是 `:3000`，后端仅本机 `:3001`。  
6. 接口 502：先查 `systemctl status ark-aigc-demo` 与 Nginx 反代。  
7. 文本对话报未配置 ApiKey：按「2.4」配置 `ARK_API_KEY` 后重启后端。
