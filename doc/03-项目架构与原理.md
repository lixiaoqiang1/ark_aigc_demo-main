# 03 - 项目架构与原理

> **本篇解决什么问题**：Node 服务端有哪些模块、怎么代理火山 API、大模型在哪里被调用。

配置字段说明见 [01-配置说明](./01-配置说明.md)。

## 一、服务端在项目中的角色

```mermaid
flowchart TB
    subgraph local [本地 Server]
        JSON[scenes/*.json]
        APP[app.js]
        UTIL[util.js]
        TOKEN[token.js]
    end
    subgraph volc [火山云端]
        API[rtc.volcengineapi.com]
        Agent[AIGC Agent]
    end
    FE[React 前端] -->|HTTP POST| APP
    JSON --> APP
    APP --> UTIL
    APP --> TOKEN
    APP -->|HMAC签名 + 转发| API
    API --> Agent
```

**核心认知**：本地服务端是「配置中心 + 签名代理 + Token 工厂」，**不直接调用 LLM SDK**。大模型在火山 AIGC-RTC 云端完成 ASR → LLM → TTS。

## 二、目录结构与模块

```
Server/
├── app.js           # Koa 入口，/proxy 和 /getScenes
├── util.js          # 读 JSON、响应封装、参数校验、ASR AppId 校验
├── token.js         # RTC AccessToken 生成
├── package.json
├── scenes/
│   └── Custom.json  # 全部业务配置
└── README.md
```

| 文件 | 职责 |
|------|------|
| `app.js` | HTTP 路由；读取场景；签名转发 OpenAPI；自动生成 RoomId/UserId/TaskId |
| `util.js` | `readFiles()`、`wrapper()`、`assert()`、`assertSpeechAppId()` |
| `token.js` | AppId + AppKey + RoomId + UserId → RTC 进房 Token |
| `scenes/*.json` | AK/SK、RTC、ASR、TTS、LLM 参数 |

```mermaid
flowchart LR
    app_js[app.js] --> util_js[util.js]
    app_js --> token_js[token.js]
    util_js --> scenes[scenes/*.json]
    app_js -->|Signer| volc_api[火山 OpenAPI]
```

## 四、两个 API

### API 1：`POST /getScenes`

```
POST http://localhost:3001/getScenes?Action=getScenes
Body: {}
```

处理流程（`app.js`）：

1. 遍历 `Server/scenes/*.json`
2. 校验 `RTCConfig.AppId`
3. 自动生成 RoomId / UserId / Token（若未填）
4. 校验 Agent UserId ≠ TargetUserId
5. 每次生成新 TaskId（UUID）
6. 剥离 `AppKey` 后返回给前端

### API 2：`POST /proxy`

```
POST http://localhost:3001/proxy?Action=StartVoiceChat
Body: { "SceneID": "Custom" }
```

| Action | body 内容 |
|--------|-----------|
| `StartVoiceChat` | 整个 `VoiceChat`（含 LLMConfig）；启动前先尝试 Stop 旧任务 |
| `StopVoiceChat` | `{ AppId, RoomId, TaskId }` |

处理：AK/SK 签名 → POST `https://rtc.volcengineapi.com?Action=xxx&Version=2024-12-01`

## 五、服务端如何「调用模型」

**不调用。** 只把 `VoiceChat.Config.LLMConfig` 转发给火山 OpenAPI。

```mermaid
sequenceDiagram
    participant FE as 前端
    participant Node as Node_Server
    participant Volc as 火山OpenAPI
    participant Cloud as AIGC云端

    FE->>Node: StartVoiceChat
    Node->>Volc: VoiceChat含LLMConfig
    Volc->>Cloud: 创建 Agent
    Cloud->>Cloud: ASR + LLM + TTS
    Cloud-->>FE: RTC 音频 + 字幕
```

LLM 配置在 `VoiceChat.Config.LLMConfig`：

| 字段 | 含义 |
|------|------|
| `Mode` | `ArkV3` = 火山方舟 |
| `EndPointId` | 方舟接入点 `ep-xxx` |
| `SystemMessages` | AI 人设 |

VoiceChat 完整结构：

| 配置块 | 作用 |
|--------|------|
| `AgentConfig` | 机器人 ID、欢迎语、目标用户 |
| `ASRConfig` | 语音识别 |
| `TTSConfig` | 语音合成 |
| `LLMConfig` | 大模型 |
| `AvatarConfig` | 数字人 |
| `InterruptMode` | 打断模式 |

## 六、RTC Token 生成

文件：`Server/token.js`

1. 输入 AppId、AppKey、RoomId、UserId
2. HMAC 签名 + Base64 打包
3. 有效期 24 小时
4. 前端 `joinRoom(token)` 进房

---

## 七、统一响应格式

`util.js` 的 `wrapper()`：

**成功**：`{ ResponseMetadata, Result }`

**失败**：`{ ResponseMetadata: { Error: { Code, Message } } }`
