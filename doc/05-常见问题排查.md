# 05 - 常见问题排查

> **本篇解决什么问题**：跑通或对话过程中遇到报错，快速定位原因和处理方法。

---

## 一、问题速查表

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| `token_error` 进房失败 | RTC AppId 与 AppKey 不匹配；或把**语音 AppId** 误填到 RTC | 见 [01 三种 ID 对照](./01-配置说明.md#二三种-id-对照最易填错) |
| 一直「AI 准备中」 | ASR/TTS AppId 填错；EndPointId 无效；云端任务僵死 | 检查语音 AppId；刷新页面；运行 restart-services.bat |
| `[getScenes] call failed` | 服务端未启动；Custom.json 缺 RTC AppId | 先启 Server；填 RTCConfig |
| `invalid TaskId` | TaskId 为空字符串 | 删除该字段或填有效值；刷新页面让服务端自动生成 |
| `targetUserID ... equal to Agent userID` | Agent UserId 与用户 UserId 相同 | RTC UserId 留空；Agent UserId 用 `ChatBot01` |
| `The task has been started` | 固定 TaskId 重复启动 | 双击 restart-services.bat；或先 Stop 再 Start |
| `Invalid 'Authorization' header` | AK/SK 错误 | 检查 AccountConfig |
| StartVoiceChat 报 ASR AppId 填错 | 用了 RTC AppId 或留空 | 填语音控制台纯数字 AppId |
| 改 JSON 后不生效 | 未刷新浏览器 | Ctrl+Shift+R |
| `Cannot read properties of undefined (reading 'app_id')` | 服务端未启动，getScenes 失败 | 先 `cd Server && python main.py`，再刷新浏览器 |
| `WinError 10013` / `EADDRINUSE 3001` | 端口已被占用 | stop-services.bat 后重试 |
| 麦克风无法开启 | 非 localhost / 非 HTTPS | 用 localhost 访问 |
| 对话无声音 | 浏览器未允许麦克风；AI 未进房 | 检查权限；看控制台 RTC 日志 |

## 二、典型问题详解

### 2.1 token_error

**原因**：RTC Token 无效。

常见填错方式：
- `RTCConfig.AppId` 填了语音 AppId（纯数字）而非 RTC AppId（24 位 hex）
- `AppKey` 与 `AppId` 不是同一 RTC 应用下的配对
- 浏览器缓存了旧 Token（改配置后未刷新）

**处理**：
1. 确认 `RTCConfig.AppId` + `AppKey` 来自 [RTC AIGC 控制台](https://console.volcengine.com/rtc/aigc/listRTC)
2. 确认 ASR/TTS AppId 在 `VoiceChat.Config` 里，**不要**放在 RTCConfig
3. 重启服务 + 硬刷新浏览器

---

### 2.2 一直「AI 准备中」

**含义**：`msgHistory` 为空，未收到云端字幕。

**原因链**：
1. StartVoiceChat 表面成功，但 ASR/TTS 配置错 → 云端 Agent 无法处理语音
2. LLM EndPointId 无效或欠费
3. 旧 TaskId 任务僵死，新会话未真正启动

**处理**：
1. 检查 ASR/TTS 用的是**语音 AppId**（纯数字）
2. 检查 `EndPointId` 为 `ep-xxx` 格式且在方舟控制台可用
3. 双击 `scripts/restart-services.bat`
4. Ctrl+Shift+R 刷新浏览器

---

### 2.3 三种 AppId 混用

```
❌ 错误：RTCConfig.AppId = "1234567890"        （这是语音 AppId）
✅ 正确：RTCConfig.AppId = "a1b2c3d4e5f6789012345678"
✅ 正确：ASRConfig.AppId  = "1234567890"
```

详见 [01-配置说明](./01-配置说明.md)。

---

### 2.4 Agent 与用户 ID 冲突

```
❌ RTCConfig.UserId = "ChatBot01"
   AgentConfig.UserId = "ChatBot01"     → 两者相同，报错

✅ RTCConfig.UserId = ""               → 留空自动生成
   AgentConfig.UserId = "ChatBot01"     → AI 机器人固定 ID
```

---

### 2.5 改配置不生效

服务端 uvicorn 会自动重启，但**前端不会自动重新请求 getScenes**。

每次改 `Custom.json` 后：
1. 等服务端重启完成（或 restart-services.bat）
2. **Ctrl + Shift + R** 硬刷新浏览器

## 三、调试命令

```powershell
# 1. 服务端是否存活
Invoke-WebRequest -Uri "http://localhost:3001/getScenes?Action=getScenes" -Method POST -ContentType "application/json" -Body "{}" -UseBasicParsing

# 2. 测试 StartVoiceChat
$body = '{"SceneID":"Custom"}'
Invoke-WebRequest -Uri "http://localhost:3001/proxy?Action=StartVoiceChat&Version=2024-12-01" -Method POST -ContentType "application/json" -Body $body -UseBasicParsing

# 3. 查看端口占用
netstat -ano | findstr ":3000"
netstat -ano | findstr ":3001"
```

| StartVoiceChat 返回 | 含义 |
|---------------------|------|
| `"Result":"ok"` | 正常 |
| `"Result":"The task has been started..."` | 旧任务未停，需 restart |
| `ResponseMetadata.Error` | 看 Message 字段 |
