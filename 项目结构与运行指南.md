# AIGC Demo 项目说明与运行指南

> 本文档基于对 `ark_aigc_demo-main` 仓库源码的逐文件梳理，整理出**项目功能结构**、**服务端与 AI 模型模块逻辑**、**完整运行命令**。
> 仓库本身是火山引擎官方「交互式 AIGC 语音场景」Demo（v1.6.0），本目录在此基础上**额外内置了两个 Python 服务端**用于二次开发。

---

## 一、项目简介

这是一个**基于流式语音的端到端 AIGC 对话 Demo**，整合了火山引擎的：

| 能力 | 全称 | 作用 |
| :-- | :-- | :-- |
| **RTC** | Real-Time Communication | 实时音视频通信，传输用户与 AI 的语音流 |
| **ASR** | Automatic Speech Recognition | 语音识别，把用户语音转成文本 |
| **LLM** | Large Language Model | 大语言模型，生成 AI 回复（豆包 / 方舟 Ark） |
| **TTS** | Text-To-Speech | 语音合成，把 AI 文本回复转成语音 |
| **RAG** | Retrieval-Augmented Generation | 检索增强，从知识库取资料喂给大模型（自定义服务端） |

用户在网页上对着麦克风说话 → RTC 采集 → 云端 ASR 转文字 → LLM 思考 → TTS 转语音 → AI 智能体在房间里"开口说话"，实现自然流畅的语音对话，可扩展数字人、视觉多模态等场景。

### 2.3 三套服务端对比（⚠️ 重点）

| 维度 | `Server/`（Node.js） | `server_python/`（Python） | `rag_llm_server/`（Python）⭐ |
| :-- | :-- | :-- | :-- |
| 框架 | Koa + nodemon | FastAPI + uvicorn | FastAPI + uvicorn |
| 端口 | **3001** | **3001** | **3001** |
| 定位 | 官方原版，标准 `ArkV3` 模式 | Node 版的 Python 复刻 | **自定义 `CustomLLM` 模式 + RAG** |
| 场景来源 | 读取 `Server/scenes/*.json` | 读取 `server_python/scenes/*.json`（**目录缺失**） | 硬编码在 `main.py` 里 |
| LLM 调用 | 云端直调豆包（ArkV3） | 同 Node（云端直调） | **回调自身 `/api/chat_callback`，RAG 检索 + 流式生成** |

| 额外依赖 | Node 生态 | fastapi/httpx | fastapi/httpx/**volcenginesdkarkruntime** |

> ⚠️ **端口冲突**：三套服务端都监听 **3001**，**同一时间只能启动其中一个**。前端 `AIGC_PROXY_HOST` 写死指向 3001，所以选哪个服务端，前端就调用哪个。

## 三、整体架构与数据流

```
┌─────────────────┐         HTTP(3001)         ┌──────────────────────────────┐
│                 │  ① POST /getScenes          │   选中的服务端（三选一）       │
│   浏览器前端     │ ─────────────────────────▶ │   Server / server_python /    │
│  (React, :3000) │  ② POST /proxy?Action=...   │   rag_llm_server   (:3001)    │
│                 │ ◀───────────────────────── │                               │
│  @volcengine/rtc│   返回场景/RTC进房参数       │   ③ 用 AK/SK 签名后转发请求    │
└────────┬────────┘                             └──────────────┬───────────────┘
         │                                                     │
         │ ④ WebRTC 音视频流（语音采集/播放）                     │ ③ 火山引擎 OpenAPI
         ▼                                                     ▼
┌────────────────────────────────────────────────────────────────────────────┐
│                  火山引擎 RTC 云端服务  (rtc.volcengineapi.com)              │
│                                                                            │
│   ASR(语音→文本) ─▶ LLM(推理) ─▶ TTS(文本→语音)  ──▶ 智能体语音回放          │
│                          │                                                 │
│                          │ ⑤ CustomLLM 模式下，LLM 推理由回调出云          │
│                          ▼                                                 │
│                ┌─────────────────────┐    RAG检索    ┌──────────────────┐  │
│                │ rag_llm_server      │ ────────────▶ │ 火山知识库 / Ark  │  │
│                │ /api/chat_callback  │               │ (api-knowledgebase│  │
│                │ (SSE 流式返回)       │ ◀──────────── │  / 豆包大模型)    │  │
│                └─────────────────────┘   流式Token    └──────────────────┘  │
└────────────────────────────────────────────────────────────────────────────┘
```

**完整调用时序**：

1. 前端启动 → 调 `POST /getScenes` 拿到**场景列表 + RTC 进房参数**（AppId/RoomId/UserId/Token）。
2. 前端用 RTC SDK 加入房间。
3. 前端调 `POST /proxy?Action=StartVoiceChat` → 服务端**签名**（AK/SK）后转发到火山 RTC OpenAPI，云端拉起一个"AI 智能体"机器人进房。
4. 用户开口说话 → RTC 采集语音流上云 → 云端 **ASR** 识别成文本。
5. 云端把文本送 **LLM**：
   - `ArkV3` 模式：云端直接调豆包，一步出结果。
   - `CustomLLM` 模式：回调 `rag_llm_server/api/chat_callback`，服务端先 **RAG** 检索知识库，再调豆包**流式生成**，按 SSE 协议返回。
6. LLM 输出文本 → 云端 **TTS** 合成语音 → 智能体在房间里播放，用户听到回复。
7. 结束时前端调 `POST /proxy?Action=StopVoiceChat` 停止任务。

## 四、服务端模块逻辑详解

### 4.1 `Server/`（Node.js 官方原版）—— `app.js`

**职责**：读场景 JSON → 签发 RTC Token → 代理火山 OpenAPI。

| 接口 | 方法 | 作用 |
| :-- | :-- | :-- |
| `/getScenes` | POST | 读取 `Server/scenes/*.json`，自动生成 RoomId/UserId/Token（若未填），返回场景 + RTC 参数给前端 |
| `/proxy?Action=StartVoiceChat` | POST | 取场景里 `VoiceChat` 整段配置，签名后转发给火山 RTC，启动智能体 |
| `/proxy?Action=StopVoiceChat` | POST | 取 AppId/RoomId/TaskId，转发停止请求 |

**核心文件**：
- `app.js`：Koa 主程序，路由 + 业务逻辑（`switch(Action)` 区分启动/停止）。
- `token.js`：**RTC Token 生成器**（HMAC-SHA256，结构 = `"001" + AppId + Base64(消息体+签名)`）。
- `util.js`：`readFiles`（读场景）、`wrapper`（统一响应封装）、`assert`（参数校验）。
- `scenes/Custom.json`：⭐ **场景配置模板，所有真实凭证填这里**。

### 4.2 `server_python/`（Python 复刻版）—— `main.py`

**职责**：与 Node 版**逻辑完全等价**的 Python 实现，接口签名一致（`/proxy`、`/getScenes`）。

**核心文件**：
- `main.py`：FastAPI 路由，逻辑与 `Server/app.js` 一一对应。
- `token_builder.py`：Token 生成（Python 版，与 `token.js` 算法相同）。
- `utils.py`：**手写的 `Signer`**，替代 Node 的 `@volcengine/openapi` 官方签名 SDK；含 `read_files`、`response_wrapper`、`assert_val`。

> ⚠️ **当前缺失**：`server_python/scenes/` 目录不存在，启动后 `getScenes` 会返回空场景列表。若要使用，需新建 `scenes/` 并放入 `Custom.json`（可参照 `Server/scenes/Custom.json`）。

### 4.3 `rag_llm_server/`（自定义 RAG + LLM 服务端）⭐ —— `main.py`

**职责**：本仓库的**核心二次开发服务**。用 `CustomLLM` 模式把 LLM 推理从云端"拉回"本地，自己实现 **知识库检索 + 大模型流式问答**。

| 接口 | 方法 | 作用 |
| :-- | :-- | :-- |
| `/getScenes` | POST | 返回**硬编码**的 Custom 场景，签发 RTC Token（固定 RoomId=`ChatRoom01`、UserId=`Huoshan01`） |
| `/proxy?Action=StartVoiceChat` | POST | 构造启动请求，`LLMConfig.Mode=CustomLLM`，`Url` 指向自身 `${SERVER_URL}/api/chat_callback` |
| `/proxy?Action=StopVoiceChat` | POST | 停止任务 |
| `/api/chat_callback` | POST | ⭐ **核心**：火山 RTC 回调入口，RAG 检索 → Ark 流式生成 → SSE 返回（OpenAI 兼容格式） |
| `/debug/chat` | POST | 调试接口：带历史对话的纯文本流式问答 |
| `/debug/rag?query=` | GET | 调试接口：单独测试知识库检索结果 |

**核心文件**：
- `main.py`：FastAPI 主程序，含 4 类接口。
- `config.py`：从 `.env` 读取所有配置（AK/SK、Ark Key/Endpoint、RTC AppId/AppKey、SERVER_URL）。
- `services/llm_service.py`：⭐ 调用**豆包（方舟 Ark）**大模型，封装 `chat_stream` 流式接口，内嵌"懂小智"人设系统提示词。
- `services/rag_service.py`：⭐ 调用**火山知识库**检索 API（`api-knowledgebase.mlp.cn-beijing.volces.com`），返回相关文档内容。
- `services/token_build.py`：Token 生成（Python 版）。
- `services/utils.py`：手写 `Signer`（与 server_python 相同算法）。
- `.env.example`：⭐ **环境变量模板**（本次新增，复制为 `.env` 后填写）。

## 五、AI 模型模块逻辑（调用链路）

### 5.1 RTC（实时音视频）

- **SDK**：前端 `@volcengine/rtc`（封装在 `src/lib/RtcClient.ts`）。
- **房间**：用户与 AI 智能体在同一个 RTC 房间，语音流在房间内双向传输。
- **鉴权**：进房需 Token，由服务端用 **AppKey + HMAC-SHA256** 算出。

### 5.2 ASR（语音识别）

- 在 `VoiceChat.Config.ASRConfig` 中配置，`Provider="volcano"`，`Mode="smallmodel"`（小模型低延迟）。
- 需填语音识别的 `AppId` 和 `Cluster`（如 `volcengine_streaming_common`）。
- 由云端 RTC 任务自动调用，开发者无需自己对接。

### 5.3 LLM（大模型）—— 两种模式

#### 模式 A：`ArkV3`（云端直调，标准 Demo 走这条路）
- `LLMConfig.Mode="ArkV3"`，填 `EndPointId`（豆包推理接入点）和 `SystemMessages`（人设）。
- 云端 RTC 直接用该 Endpoint 调豆包，无需本地 LLM 服务。
- `Server/scenes/Custom.json` 默认就是这种模式。

#### 模式 B：`CustomLLM`（自定义回调，rag_llm_server 走这条路）⭐
- `LLMConfig.Mode="CustomLLM"`，填 `Url`（指向自己的 HTTP 接口）、`Method`、`ApiType`。
- 云端每次需要 LLM 推理时，**回调**该 Url，把对话消息 POST 过来。
- 本地服务（`/api/chat_callback`）流程：
  1. 接收 `messages`（最后一条是用户最新发言）。
  2. **RAG 检索**：`rag_service.retrieve(用户问题)` → 从火山知识库取相关文档。
  3. **拼系统提示词**：人设 + `### 参考知识库` + 检索内容。
  4. **流式调用豆包**：`llm_service.chat_stream()` → Ark SDK `stream=True`。
  5. **SSE 返回**：每个 chunk 包成 `data: {json}\n\n`，结尾发 `data: [DONE]\n\n`（符合 RTC 要求的 OpenAI 兼容流式格式）。

> ⚠️ `CustomLLM` 模式下，`Url` 必须是**火山云端能访问到的公网地址**，本地调试需用内网穿透（ngrok / cpolar / frp）暴露 `SERVER_URL`。

### 5.4 TTS（语音合成）

- 在 `VoiceChat.Config.TTSConfig` 中配置，`Provider="volcano"`。
- `voice_type` 选音色（如 `BV001_streaming`），可调 `speed_ratio`/`pitch_ratio`/`volume_ratio`。
- 由云端自动调用，把 LLM 文本转成智能体语音。

### 5.5 RAG（检索增强）—— 仅 rag_llm_server

- `rag_service.py` 调用火山**知识库服务**（Service=`air`，Region=`cn-north-1`）。
- 请求 `POST /api/knowledge/collection/search_knowledge`，Header 必须带 `V-Account-Id`。
- 用 AK/SK 走同样的 HMAC-SHA256 签名。
- 返回 `result_list`，提取每条 `content` 拼成上下文，注入 LLM 系统提示词。

### 5.6 鉴权密钥体系（重要）

| 密钥 | 用途 | 配置位置 |
| :-- | :-- | :-- |
| **AK / SK**（VOLC_ACCESS_KEY / VOLC_SECRET_KEY） | 签名调用火山 OpenAPI（RTC / 知识库） | `scenes/*.json` 的 AccountConfig 或 `.env` |
| **AppKey**（RTC_APP_KEY） | 生成 RTC 进房 Token | `scenes/*.json` 的 RTCConfig 或 `.env` |
| **Ark API Key + Endpoint ID** | 调用豆包大模型 | 仅 rag_llm_server 用，配在 `.env` |

## 六、运行命令

### 6.1 环境要求

| 工具 | 要求 | 当前机器检测 |
| :-- | :-- | :-- |
| Node.js | 16.0+（推荐 18/20） | ✅ v20.10.0 |

### 6.2 选择运行哪套服务端

> **三套服务端二选一**（都占 3001 端口）。新手先跑 **`Server/`（Node 版）** 最简单。

| 目标 | 推荐服务端 | 原因 |
| :-- | :-- | :-- |
| 还原官方 Demo | `Server/`（Node） | 文档全、签名用官方 SDK、配 JSON 即可 |
| 想要 RAG + 自定义大模型问答 | `rag_llm_server/`（Python） | 知识库增强 + 人设 + 流式 |
| 只想要 Python 版官方逻辑 | `server_python/` | 需自备 scenes 目录 |

### 6.3 通用前置：配置场景凭证

无论哪套服务端，都需要**真实火山引擎凭证**才能跑通完整对话链路（涉及付费服务 ASR/TTS/LLM/RTC）：

1. 开通服务：<https://www.volcengine.com/docs/6348/1315561>
2. 获取 AK/SK：<https://console.volcengine.com/iam/keymanage/>
3. 获取 RTC AppId/AppKey：<https://console.volcengine.com/rtc/aigc/listRTC>
4. 填入对应配置文件（见下方各服务端小节）。

> ⚠️ **不填凭证也能启动服务进程**，但调用 `/getScenes` 或 `/proxy` 会因校验失败报错；前端会卡在"AI 准备中"。

## 八、常见问题速查

| 问题 | 解决方案 |
| :-- | :-- |
| 前端编译报 `craco: command not found` | `npm install` 未完成，删 `node_modules` 重装 |
| 服务端 `EADDRINUSE :::3001` | 3001 端口被占，**别同时开两套服务端**；`netstat -ano \| findstr :3001` 查进程 |
| `/proxy` 返回 `Invalid 'Authorization' header` | AK/SK 不正确或未填 |
| 浏览器报 `token_error` | RTC Token 不合法/过期，检查 AppKey、RoomId、UserId 是否与生成时一致 |
| 一直卡在"AI 准备中" | 多半是凭证未开通/欠费/参数大小写错误；逐项核对 ASR/TTS/LLM/RTC 配置 |
| `StartVoiceChat` 报 `task has been started` | 固定 RoomId/UserId 重复启动，先调 StopVoiceChat 再 StartVoiceChat |
| 麦克风/摄像头开不了 | 页面必须是 localhost 或 https（安全上下文） |
| rag_llm_server 启动报 `No module named 'volcenginesdkarkruntime'` | `pip install volcenginesdkarkruntime`（requirements.txt 漏装） |
| rag_llm_server 对话无回复 | `SERVER_URL` 不是公网地址，火山云端回调不到；用内网穿透 |
| 想接自己的服务端 | 改 `src/config/index.ts` 的 `AIGC_PROXY_HOST` 和 `src/app/api.ts` 的 `APIS_CONFIG` |
